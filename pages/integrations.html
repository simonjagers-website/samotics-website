Integration in practice

From detection to maintenance action — automatically

Integration isn't about connecting systems. It's about closing the loop between "we detected a fault" and "we fixed it." Here's what that looks like.

1

SAM4 detects a fault

AI identifies an anomaly in the electrical signature. A Samotics diagnostician validates the fault type, severity, and recommended action.

2

Alert pushes to your CMMS

The validated alert — including fault type, affected asset, severity, and recommended action — flows directly into your CMMS as a notification or work order.

3

Maintenance team acts

The work order lands in the existing maintenance workflow. No new system to learn. No portal to check. The fault information is where your team already works.

CMMS integration

SAP PM. IBM Maximo. Your existing workflows.

The two most common CMMS platforms in heavy industry. SAM4 integrates with both — plus any system that accepts API input.

SAP Plant Maintenance

SAM4 alerts create notifications or maintenance orders in SAP PM. Asset mapping links SAM4 assets to your SAP functional locations. Alert data populates the order description, priority, and recommended action — ready for the planner.

Notification type, priority mapping, and field configuration are set during onboarding.

IBM Maximo

SAM4 creates work orders in Maximo with fault classification, asset reference, and diagnostic context. The integration maps SAM4 severity levels to Maximo priority codes. Your maintenance planners see the alert in their standard queue.

Supports Maximo 7.6+ and Maximo Application Suite (MAS).

Other CMMS platforms

Any CMMS that accepts REST API input can receive SAM4 alerts. The integration layer is standardised — Samotics configures the mapping between SAM4 alert fields and your system's work order schema during deployment.

Custom CMMS integrations typically configured within 1–2 weeks.

SCADA & API

Connections beyond CMMS

CMMS handles the maintenance loop. But some customers need SAM4 data in SCADA dashboards, custom BI tools, or automated workflows. The REST API makes that possible.

SCADA connectivity

For Water and Oil & Gas customers who operate from SCADA control rooms, SAM4 asset health data can surface alongside existing process data. Operators see pump health next to flow rate and pressure — without switching systems.

REST API

Full programmatic access to SAM4 data: asset health scores, alert history, diagnostic reports, and fleet analytics. Use it for custom dashboards, data exports, automated workflows, or integration with any system in your stack.

Email & notification channels

For teams that don't use a CMMS, alerts push to email, SMS, or webhook endpoints. Each alert includes the full diagnostic context — fault type, severity, asset, and recommended action — so the recipient can act immediately.

Data export & reporting

Export fleet data for regulatory reporting, quarterly reviews, or capital planning. Scheduled exports or on-demand via the API. Formats include CSV, PDF reports, and structured JSON for programmatic consumption.
